## 1. 共通前提条件

- [ ] OBS Studio 28 以降・コナステ pop'n music lively・LivelyRec が起動できる
- [ ] LivelyRec で OBS 接続・記録開始ができる（結合テスト IT-OBS-01〜03 が合格済み）
- [ ] 設定「OBS」タブでソース名が「OBSから取得」で選択済み

---

## 2. テストデータ収集（区分A 認識精度測定用）

**目的**: 認識正答率を測定するための、正解ラベル付き実プレイフレームを収集する。

**手順**:

1. LivelyRec の「設定」→「記録」タブで **「デバッグ撮影」を ON** にして保存する。
2. 記録を開始し、pop'n music lively を**数曲プレイ**する（選曲→準備→プレイ→リザルトを通す）。
3. 記録を停止する。`livelyrec_data/debug/` にフレーム画像（`時刻_画面種別.png`）が保存される。
4. 各画面**30 枚以上**を目安に集まるまで 1〜3 を繰り返す。
   - **数字「7」「9」を含むリザルト**（スコア・判定数）を必ず含めること（R-022）。
   - ロード画面・画面切替の瞬間のフレームも含まれてよい（画面誤判定の検証に使う）。
5. 収集したフレームの**正解値**を付与する。`tests/fixtures/sample/_ground_truth.csv` に追記:
   ```
   path,screen,song_id,score,cool,great,good,bad,combo
   ```
   - `path`: `tests/fixtures/sample/` からの相対パス（フレームを `tests/fixtures/sample/` 配下へコピーする）
   - `screen`: 画面種別（select/play/result 等）
   - リザルトフレームは score/cool/great/good/bad/combo を、プレイフレームは score を記入
   - 不明・非該当のセルは空欄で可

> 正解値の付与が難しい場合は、フレーム画像をそのまま共有いただければ開発側でラベル付けを補助します。

## 3. 連続稼働テスト

**目的**: 1配信相当の連続稼働でクラッシュ・データ欠損が無いことを確認する。

**手順**:

1. OBS・ゲーム・LivelyRec を起動し、記録を開始する。
2. **1 時間以上**（可能なら配信相当）連続でプレイ・記録する。
3. 稼働中、以下を観察する:
   - アプリ・OBS がクラッシュ／フリーズしないか
   - LivelyRec のメモリ使用量が増加し続けないか（タスクマネージャで確認）
4. 記録停止後、「直近リザルト」パネルおよび CSV エクスポートで、プレイした曲数とリザルトが**欠損・重複なく**記録されているか確認する。

**期待結果**:

- 連続稼働中にクラッシュ・フリーズが発生しない。
- 記録データに欠損・重複が無い。
- メモリ使用量が増加し続けない。

**結果記録欄**:

## 4. 配信支援表示テスト

**目的**: ブラウザソースが記録に追従して表示更新されることを確認する。

**手順**:

1. LivelyRec を起動し、配信支援URL `http://127.0.0.1:14514/browser/index.html` を OBS ブラウザソース（または通常ブラウザ）で開く。
2. 記録を開始し、プレイ→リザルトまで進める。
3. ブラウザソース上の楽曲名・ベストスコア・直近リザルト等の表示を観察する。

**期待結果**:

- 楽曲選択・リザルト確定がブラウザソースに反映される。
- 表示更新の体感遅延が大きすぎない（目安 1 秒以内）。

## 5. 業務日切替テスト

**目的**: 業務日切替（既定 AM6:00）が正しく動作することを確認する。

**手順**:

1. 設定「記録」タブで業務日切替時刻を、検証しやすい時刻（例: 数分後）に一時変更する。
2. 記録を開始し、切替時刻をまたぐ。
3. 「業務日カウンタ」パネルの業務日表示・打鍵カウンタがリセットされるか確認する。
4. 確認後、切替時刻を既定（6時）に戻す。
