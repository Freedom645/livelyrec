## 1. pop'n music 固有用語

| 用語                | 読み／略称                      | 説明                                                                                        |
| ------------------- | ------------------------------- | ------------------------------------------------------------------------------------------- |
| pop'n music lively  | ポップンミュージック ライブリー | KONAMI Amusement のコナステ配信版 pop'n music。本アプリの対象タイトル                       |
| コナステ            | —                               | KONAMI Amusement Service Station の略。PC からアーケード音ゲーをプレイできる配信サービス    |
| e-amusement / e-amu | —                               | KONAMI の認証・課金プラットフォーム。コナステ含む各種サービスのアカウント基盤               |
| 譜面                | ふめん                          | 楽曲×難易度の組合せ。「同一楽曲でもEASY/NORMAL/HYPER/EX/UPPERは別譜面」と扱う               |
| UPPER譜面           | アッパーふめん                  | 同一楽曲に対して追加で用意された別譜面。判別が必要                                          |
| ジャンル            | —                               | pop'n では楽曲の「ジャンル名」が主タイトル扱いで表示される（例: HARDCORE TECHNO、J-POP 等） |

### 1.1. 難易度（Difficulty）

| 用語                      | 説明                                                       |
| ------------------------- | ---------------------------------------------------------- |
| EASY / 5BUTTONS / 5ボタン | 入門難易度                                                 |
| NORMAL                    | 通常難易度                                                 |
| HYPER                     | 中級難易度                                                 |
| EX / EXTRA                | 上級難易度                                                 |
| UPPER                     | 追加実装された差分譜面（譜面難易度ラベルとは独立した属性） |

### 1.2. 画面（Screens）

| 画面名         | 説明                                       | スコア記録対象         |
| -------------- | ------------------------------------------ | ---------------------- |
| 選曲画面       | 楽曲を選ぶ画面。カーソル上の譜面情報を取得 | 記録対象外（参照のみ） |
| 準備画面       | 譜面決定後の確認画面                       | 記録対象外             |
| オプション画面 | 設定変更画面（オプション）                 | 記録対象外             |
| プレイ画面     | 実際にプレイする画面                       | 打鍵数累計のソース     |
| リザルト画面   | プレイ終了後のスコア表示画面               | 主要記録ソース         |
| ロード画面     | 画面遷移時のロード                         | 状態遷移検知に用いる   |

### 1.3. スコア・判定

| 用語                      | 説明                                                                                               |
| ------------------------- | -------------------------------------------------------------------------------------------------- |
| スコア                    | プレイ結果の数値。0〜100,000 の整数                                                                |
| COOL                      | 最高判定。色: 赤紫（マゼンタ系）                                                                   |
| GREAT                     | 第2判定。色: 黄                                                                                    |
| GOOD                      | 第3判定。色: 赤                                                                                    |
| BAD                       | 失敗判定。色: 水色（シアン系）                                                                     |
| コンボ数                  | 連続成功した判定の最大数                                                                           |
| 判定数                    | 各判定（COOL/GREAT/GOOD/BAD）の発生回数。プレイ画面・リザルト画面双方で表示                        |
| 打鍵数（総打鍵数）        | LivelyRec がプレイ日単位で累計するキーアクション総数。判定数の総和に相当                             |
| 判定別打鍵数              | 判定区分（COOL/GREAT/GOOD/BAD）ごとに累計した打鍵数                                                |
| クリア種類                | リザルト画面でゲームが提示するクリア状態。LivelyRec は4種（PERFECT/FULL COMBO/CLEAR/FAILED）を扱う |
| ASSIST CLEAR / EASY CLEAR | 画面内では判別不可。LivelyRec のスコープ外                                                         |
| クリアメダル              | プレイ結果に応じたメダル種別。星／ダイヤ／丸の3種。詳細は外部Wiki参照                              |
| クリアランク              | スコアとクリア成否で決まるランク。S, AAA, AA, A, B, C, D, E の8種（クリア失敗時は S/AAA/AA は付かず A 止まり） |

## 2. アプリケーション用語

| 用語                   | 説明                                                                                      |
| ---------------------- | ----------------------------------------------------------------------------------------- |
| LivelyRec              | 本プロダクト名（Lively + Recorder）                                                       |
| プレイ日               | アプリ内で記録単位として用いる「日」。**AM 6:00** を切替時刻とする。旧称「業務日」（UI 表示・要件文言を 2026-05-22 に変更）。コード内部識別子は `business_date` のまま |
| プレイ日切替時刻       | デフォルト AM 6:00。AM 5:00〜7:00 のメンテナンスに被らない時間として採用                  |
| 記録中 / 記録停止      | 本アプリの主要な状態。記録中は OBS と接続し画面判別を継続する                             |
| 接続中                 | OBS への接続を試みている、または再接続待機している状態                                    |
| 未特定（楽曲）         | 楽曲名OCR・マスタ照合のいずれかで一意に特定できなかった状態                               |
| マスタデータ           | 楽曲一覧データ。GitHub Pages 等で配布する JSON                                            |
| 配信支援ブラウザソース | OBS の「ブラウザソース」機能で表示する HTML ページ。LivelyRec から WebSocket でデータ受信 |
| 外部連携WebSocket      | LivelyRec が公開する WebSocket API。外部ツール向け                                        |

## 3. 技術用語

| 用語                   | 説明                                                                           |
| ---------------------- | ------------------------------------------------------------------------------ |
| OBS Studio             | オープンソースの配信ソフトウェア。本アプリの画面取得元                         |
| OBS WebSocket          | OBS Studio のリモート制御プロトコル。v5（同梱）を前提とする                    |
| ブラウザソース         | OBS の機能の一つ。任意のURLをWebブラウザレンダリング結果としてシーンに合成可能 |
| WebSocket              | アプリ間・ブラウザ間の双方向通信プロトコル                                     |
| OCR                    | Optical Character Recognition。画像から文字を抽出する技術                      |
| Tesseract              | OSS の OCR エンジン                                                            |
| PaddleOCR              | 中国製の高精度 OCR ライブラリ。日本語対応                                      |
| OpenCV                 | 画像処理ライブラリのデファクト                                                 |
| テンプレートマッチング | 画像内に既知のテンプレート画像と一致する領域を検出する手法                     |
| 二値化                 | 画像をしきい値で白黒に変換する処理                                             |
| HSV変換                | 色相・彩度・明度に基づく色空間変換。色ベースの判定抽出に有効                   |
| SQLite                 | 軽量な組込み RDBMS。本アプリの永続化候補                                       |
| Semantic Versioning    | バージョン番号体系 `MAJOR.MINOR.PATCH`                                         |
| DPAPI                  | Windows のデータ保護API。ユーザコンテキストでの暗号化に用いる                  |
