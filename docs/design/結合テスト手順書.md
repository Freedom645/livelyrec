---

## 1. 共通前提条件

実施前に以下を準備すること。

- [ ] Windows 10/11 の実機
- [ ] OBS Studio 28 以降（WebSocket Server 機能を含む版）をインストール済み
- [ ] OBS の「ツール → WebSocket サーバー設定」で **サーバーを有効化**し、ポート（既定 4455）とパスワードを控えておく
- [ ] OBS に pop'n music lively の画面を映すソース（ゲームキャプチャ／ウィンドウキャプチャ等）を1つ作成し、ソース名を控えておく
- [ ] コナステ pop'n music lively を起動できる環境
- [ ] LivelyRec（本アプリ）をビルド済み、または `python -m livelyrec.app` で起動できる状態
- [ ] インターネット接続（PaddleOCR モデルの初回ダウンロードに必要）

---

## 2. テスト手順

各シナリオの「結果記録欄」に、実施日・結果（合格／不合格）・備考を記入する。

### IT-OBS-01: 実 OBS Studio への接続

**目的**: `obs_client` が実 OBS と WebSocket v5 で疎通できることを確認する。

**手順**:

1. OBS Studio を起動し、WebSocket サーバーが有効であることを確認する。
2. LivelyRec を起動する。
3. 「設定」→「OBS」タブで、ホスト・ポート・パスワード・ソース名を入力し保存する。
4. メイン画面の「記録開始」を押す。

**期待結果**:

- 接続パネルの状態表示が「接続中…」→「記録中（画面確定待ち）」または「記録中」へ遷移する。
- アプリがクラッシュしない。

| ---------- | ------ | ---- | --------------------------------------------------------- |
| 2026/05/20 | 開発者 | NG   | 接続中のまま止まってしまった。OBSはUIが固まってしまった。 |
| 2026/05/20 | 開発者 | OK   |                                                           |

### IT-OBS-02: ソースのスクリーンショット取得

**目的**: 指定ソースから 1366x768 の PNG キャプチャを取得できることを確認する。

**手順**:

1. IT-OBS-01 の状態（記録中）から、OBS のプレビューに pop'n music lively の任意の画面を表示する。
2. LivelyRec の「現在状態」パネルを観察する。

**期待結果**:

- 「画面」欄が「未確定」以外（選曲画面・プレイ画面 等）に変化する。
  → スクリーンショットが取得され画面判別が動作していることを示す。

| 実施日     | 実施者 | 結果 | 備考                                                                             |
| ---------- | ------ | ---- | -------------------------------------------------------------------------------- |
| 2026/05/20 | 開発者 | OK   | ロード画面や定義した画面以外で誤った判定がされるなどの精度が甘い様子が見られる。 |

### IT-OBS-03: 接続失敗時のエラーハンドリング

**目的**: OBS 未起動・誤パスワード時に、クラッシュせずエラーを提示することを確認する。

**手順**:

1. OBS Studio を**終了**した状態で LivelyRec の「記録開始」を押す。
2. 次に OBS を起動し、LivelyRec の設定で**誤ったパスワード**を入力して「記録開始」を押す。

**期待結果**:

- いずれの場合もアプリはクラッシュせず、状態表示が「接続中…」のまま、または「未接続」へ戻る。
- ログ（`livelyrec_data/logs/`）に接続失敗の記録が残る。

| 実施日     | 実施者 | 結果 | 備考 |
| ---------- | ------ | ---- | ---- |
| 2026/05/20 | 開発者 | OK   |      |

### IT-BROWSER-01: 配信支援ブラウザソースの表示

**目的**: WebSocket Server とブラウザソースが連携し、表示が更新されることを確認する。

**手順**:

1. LivelyRec を起動し、メイン画面の「配信支援URL」パネルからブラウザソース URL を控える。
2. OBS にブラウザソースを追加し、その URL を設定する（または通常のブラウザで URL を開く）。
3. LivelyRec で記録を開始し、pop'n music lively をプレイしてリザルト画面まで進める。

**期待結果**:

- ブラウザソース上に判定数・直近リザルト等が表示・更新される。
- 表示更新の体感遅延が大きすぎない（目安 1 秒以内）。

| 実施日     | 実施者 | 結果 | 備考                                                                                                                                                                           |
| ---------- | ------ | ---- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 2026/05/20 | 開発者 | NG   | Failed to open a WebSocket connection: invalid Connection header: keep-alive. You cannot access a WebSocket server directly with a browser. You need a WebSocket client.が表示 |
| 2026/05/20 | 開発者 | OK   | アプリ本体の判定数などのカウンタが動いていないが選曲とベストスコアは確認できたためOKとする                                                                                     |

### IT-OCR-01: PaddleOCR 実モデルでの認識

**目的**: PaddleOCR の学習済みモデルがロードされ、楽曲名・数値の OCR が動作することを確認する。

**手順**:

1. LivelyRec を初回起動環境で動かす（PaddleOCR モデルが未ダウンロードの状態）。
2. 記録を開始し、pop'n music lively のプレイ画面（上部に楽曲名が表示される画面）を映す。
3. 初回はモデルのダウンロードで数十秒待つ場合がある。

**期待結果**:

- モデルのダウンロード・ロードが完了する（ログに記録）。
- 「現在状態」パネルの「楽曲」欄に楽曲名が表示される（多少の誤認識はマスタ照合で吸収される）。

| 実施日     | 実施者 | 結果 | 備考                                                                                                                                                           |
| ---------- | ------ | ---- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 2026/05/20 | 開発者 | NG   | ログにモデルダウンロードの結果が表示されない。また、選曲画面の認識はするが楽曲名などはハイフン表示のままとなる。                                               |
| 2026/05/20 | 開発者 | NG   | 楽曲名の表示および精度はかなり低いがスコアの記録を確認した。スコアの精度が悪い原因は表示しきる前に判定していることが原因と思われる。判定数は記録されなかった。 |

---

### IT-FULL-01: 実環境でのアプリ起動〜記録開始/停止

**目的**: 実環境一式でアプリが起動し、記録の開始・停止が安定して行えることを確認する。

**手順**:

1. OBS Studio・コナステ pop'n music lively・LivelyRec をすべて起動する。
2. LivelyRec で記録を開始する。
3. 数曲プレイし、リザルトが「直近リザルト」パネルに表示されることを確認する。
4. 記録を停止し、再度開始できることを確認する。
5. アプリを終了する。

**期待結果**:

- 記録の開始・停止・再開がクラッシュなく行える。
- プレイしたリザルトが記録され UI に反映される。
- 終了時に異常終了しない。

| 実施日     | 実施者 | 結果 | 備考                                                                   |
| ---------- | ------ | ---- | ---------------------------------------------------------------------- |
| 2026/05/20 | 開発者 | NG   | 画面判別は問題なく動作するが、各項目の画像認識が動かず記録がされない。 |
| 2026/05/20 | 開発者 | OK   | 記録、リザルト表示、停止、再開は問題ないため、本ケースはOKとした。     |
